# 初始化引导提示词

> 读者：第一次安装或迁移能力库时的 AI。
> 用法：把 `<能力库根目录>` 替换为本机真实路径，复制给 AI。

---

我现在要初始化一个本地能力库。根目录是：

```text
<能力库根目录>
```

请按以下步骤执行。

## 1. 读取必要文档

先读取：

```text
CAPABILITY.md
docs/能力库产品架构.md
```

需要新增、修改、迁移能力时，再读取：

```text
skills/capability-library-maintenance/SKILL.md
```

## 2. 确认命令环境

所有命令在能力库根目录下运行。Python 命令按系统选择：

- Windows：优先 `python`，不行再试 `py -3`
- macOS/Linux：优先 `python3`，不行再试 `python`

如果 Python 不可用，告知用户，不要继续猜。

## 3. 盘点当前平台能力

盘点当前 AI 平台已配置的 Skill 和 MCP。如果无法自动列出，向用户说明需要其提供。

## 4. 迁移或同步能力

### 迁移 Skill

1. 检查 `skills/` 下是否已有相同或相近内容。
2. 保持用户原有分组习惯。
3. 创建或更新对应 `SKILL.md`。
4. 同步更新 `CAPABILITY.md`。

### 迁移 MCP

1. 每次只迁移一个 MCP。
2. 转换为 `mcps/registry.json` 的统一格式。
3. 同步更新 `CAPABILITY.md`。
4. 运行 `load_mcp.py` 测试，确认 MCP 可用。

禁止在迁移后自动删除原平台配置。只有用户确认后才可以删除。

## 5. 初始化自测

至少运行本地回声 MCP 的加载测试：

```powershell
python tools/mcp/load_mcp.py --name "本地回声 MCP"
```

再做最小调用确认链路正常：

```powershell
python tools/mcp/use_tool.py --mcp "本地回声 MCP" --tool "echo" --params-json "{\"text\": \"hello\"}"
```

如果失败，把错误原因告诉用户。

## 6. 写入长期使用说明

初始化完成后，把以下内容写入当前 AI 平台的全局记忆或启动上下文：

1. 将CAPABILITY.md通过软链接或类似方案，注入到AI Agent系统的全局初始上下文中，具体方案取决于使用的平台。例如Claude Code可以在全局CLAUDE.md文件内使用@方案进行链接，codex可以将全局AGENTS文档软链接到目标文件。

如果不能使用软链接将文档完全链接，至少也应当将能力库的项目路径、CAPABILITY文档的内容完整写入对应的记忆文件。务必让AI在初始上下文就获取关于能力库的整体信息，以提高AI调用能力库的可能。

2. 用系统内置的skill创建方案创建一个“能力库维护skill”，按照所在平台要求写清楚skill名称、元信息等。skill正文只有一句话：

```text
本skill是索引，完整内容在“<能力库根目录>/skills/capability-library-maintenance/SKILL.md”
```

如果用户要求，也可以将文档内容直接写入系统内置的skill。

说明：通过软链接、文档索引可以实现一处修改全局生效，只有在软链接或索引有重大缺陷时，才直接将文档内容写入系统内部。
