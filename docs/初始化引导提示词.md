# 初始化引导提示词

> 以下内容是一个可直接发给 AI 的提示词。
> 复制全部内容，粘贴到 Hermes、Claude Code 等任意 AI 平台，AI 将自动完成能力库的初始化。

---

我现在开始使用一个叫"能力库"的本地工具。它是一个跨 AI 平台的 skill 和 MCP 注册与调用系统。项目路径："D:\path\to\capability-library"【这里应该替换为你的实际目录】

## 请你按以下步骤操作

### 第一步：了解能力库

读取以下两个文件，理解能力库的机制：

1. `skills/capability-library-guide/SKILL.md` — 能力库使用说明
2. `skills/intake-flow/SKILL.md` — 入库流程

### 第二步：写入长期运行时入口

读取 `CAPABILITY_LIBRARY_CONTEXT.md`，将其中的示例路径替换为本机实际能力库路径，并把全部内容写入当前 AI 平台的全局记忆或启动上下文。

这样 AI 在任何新会话中都会知道能力库的存在和使用方法，而不会让用户精心积累的能力被AI忽略。

注意：`CAPABILITY_LIBRARY_CONTEXT.md` 是每次启动会话都应该让AI知道的使用指南，不应作为普通业务 Skill 依赖触发（普通业务skill只会加载一句话的skill描述，内容不够完整）。

### 第三步：配置 API Key

复制 `config.example.json` 为 `config.local.json`，填入 DeepSeek API Key（用于前台咨询）。

### 第四步：盘点当前 MCP 和 skill

查看当前 AI 平台已配置的 MCP 服务列表和 skill 列表。

### 第五步：逐个迁移入库

对于每个 MCP：
- 按照 `skills/intake-flow/SKILL.md` 中的入库流程
- 逐一将 MCP 配置转换为能力库统一格式
- 逐一运行 `register_mcp.py` 注册
- 务必每次只注册一个，注册成功后再注册下一个。

对于每个 skill：
- 将 skill 文件复制到 `skills/` 下
- 运行 `register_skill.py` 注册
- 尽可能保持用户已有的分组方案，不要把所有skill都放在同一级目录中。

### 第六步：更新全局记忆

在 AI 平台的全局记忆中记录：

```text
每次新对话开始时，在处理非简单任务前，先读取 <能力库实际路径>\CAPABILITY_LIBRARY_CONTEXT.md。使用该文件了解如何发现本地 Skill 和 MCP；不确定该用什么能力时，调用 ability_suggest.py。
```

---

## 备注

- 迁移完成后，可从 AI 平台配置中删除已迁移的 MCP 和 skill，由能力库统一管理【特别提示，这一步必须要先向用户确认，严禁在迁移后自动删除配置，因为用户还需要先测试、习惯】。
- 能力库的 `config.local.json` 只需填 DeepSeek Key，MCP 的 Key 直接写在 `mcps/registry.json` 里。
- 想知道更多设计细节可读 `docs/能力库设计思路.md`。
