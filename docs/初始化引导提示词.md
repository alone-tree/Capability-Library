# 初始化引导提示词

> 读者：第一次安装或迁移能力库时的 AI。
> 用法：把本文件复制给 AI，并先把 `<能力库根目录>` 替换成本机真实路径。
> 目标：把当前平台已有的 Skill 和 MCP 迁移到能力库，并让后续新会话知道必须使用能力库。

---

我现在要初始化一个本地能力库。能力库根目录是：

```text
<能力库根目录>
```

请按以下步骤执行。

## 1. 读取必要文档

先读取：

```text
docs/能力库使用说明.md
skills/intake-flow/SKILL.md
```

`docs/能力库使用说明.md` 是后续每次新会话都应知道的长期使用说明。  
`skills/intake-flow/SKILL.md` 是迁移 Skill/MCP 时使用的入库流程。

## 2. 确认命令环境

所有命令都在能力库根目录下运行。

进入目录：

```bash
cd "<能力库根目录>"
```

Python 命令按当前系统选择：

- Windows 优先尝试：`python`，不行再试 `py -3`
- macOS / Linux 优先尝试：`python3`，不行再试 `python`

如果 Python 命令不可用，先告知用户，不要继续猜。

## 3. 配置 advisor

如果不存在 `config.local.json`，复制 `config.example.json` 为 `config.local.json`，并让用户填入 DeepSeek API Key。

不要替用户编造 API Key。

## 4. 盘点当前平台能力

盘点当前 AI 平台已经配置的：

- Skill
- MCP

你可以从当前平台的配置文件中读取相应的内容。如果当前平台无法自动列出这些内容，向用户说明需要其提供配置位置或清单。

## 5. 逐个迁移入库

迁移 Skill：

1. 保持用户原有分组习惯。
2. 把每个 Skill 放到 `skills/` 下的独立目录。
3. 必须遵照 `skills/intake-flow/SKILL.md` 中的注册流程。

迁移 MCP：

1. 每次只迁移一个 MCP。
2. 转换为能力库统一格式。
3. 使用 `skills/intake-flow/SKILL.md` 中的注册流程
4. 一个成功后再处理下一个。

禁止在迁移后自动删除原平台配置。只有用户明确确认后，才可以协助删除。

## 6. 初始化自测

构思一个典型案例，在能力库根目录下运行一次完整流程，测试相应的配置是否生效：

1、向advisor提出一个任务咨询，预期会收到一个json格式的返回。
2、收到advisor的建议，读取skill/加载MCP，获取相应说明
3、调用MCP的指定工具，得到返回结果。

如果当前系统使用 `python3` 或 `py -3`，替换上面命令里的 `python`。

如果失败，把错误原因告诉用户。

## 7. 写入长期使用说明

在一切配置完成后，修改 `docs/能力库使用说明.md` 中的 `<能力库根目录>` 替换成本机真实路径。把能力库涵盖范围替换为真实的内容。如果使用说明中的命令有误，需要替换为实际可用的命令。

然后将替换后的全文写入当前 AI 平台的全局记忆、启动上下文或等价机制。

如果平台记忆容量有限，至少写入一下内容，让后续AI知道有能力库的存在：

```text
每次新对话开始时，在处理非简单任务前，先读取 <能力库根目录>/docs/能力库使用说明.md。该文件说明如何发现和调用本地 Skill 与 MCP。
```
