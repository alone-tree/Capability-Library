# 初始化引导提示词

> 读者：第一次安装或迁移能力库时的 AI。

能力库根目录由用户指定。先运行：

```powershell
python scripts/deploy.py <能力库根目录> --check
python scripts/deploy.py <能力库根目录>
```

部署脚本会创建缺失的初始模板，但不会覆盖已有的 `capability-entry/`、`skills/`、`mcps/registry.json` 或 `CAPABILITY.md`。

## 初始化能力地图

读取 `capability-entry/SKILL.md`，把用户正式保留的低频 Skill 组和 MCP 摘要写入该入口。具体 Skill 正文仍保存在 `skills/`，MCP 的真实配置仍保存在 `mcps/registry.json`。

平台本地 Skill 根目录保持为普通目录，只把 `capability-entry` 链接到能力库入口。用户通用 Skill 使用独立共享目录，不把整个能力库链接进平台。

## 初始化 MCP

按照 `docs/MCP使用指南.md` 先加载本地回声 MCP：

```powershell
python tools/mcp/load_mcp.py --name "本地回声 MCP"
```

再执行最小调用：

```powershell
python tools/mcp/use_tool.py --mcp "本地回声 MCP" --tool "echo" --params-json "{\"text\": \"hello\"}"
```

默认 load 不保活。只有连续复用 stdio MCP 时才使用 `--keep-alive`，并在任务完成后执行 `--close`。

## 迁移现有能力

每次只迁移一个 Skill 组或 MCP。迁移后同步更新 `capability-entry/SKILL.md`，并完成安全的最小 load/use 验证。未经用户确认，不删除原平台配置或旧文件。
