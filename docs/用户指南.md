# 能力库用户指南

## 1. 一分钟快速开始

### 1.1 安装

1. 下载 `capability-library.zip` 并解压到任意目录
2. 复制 `config.example.json` 为 `config.local.json`
3. 编辑 `config.local.json`，填入你的 API Key

```json
{
  "deepseek": {
    "api_key": "sk-你的DeepSeek-API-Key",
    "base_url": "https://api.deepseek.com/chat/completions",
    "model": "deepseek-v4-flash"
  }
}
```

MCP 的 API Key 和 Token 直接写在 `mcps/registry.json` 中。config.local.json 只管 DeepSeek。

### 1.2 验证安装

```powershell
# 测试前台咨询（需要 DeepSeek Key）
python tools/advisor/ability_suggest.py "帮我搜索资料"

# 测试本地回声 MCP（无需任何 Key）
python tools/mcp/load_mcp.py --name "本地回声 MCP"
python tools/mcp/use_tool.py --mcp "本地回声 MCP" --tool "echo" --params-json "{\"text\":\"hello\"}"
```

## 2. 核心概念

能力库把 AI 需要的外部能力分成两类，统一管理：

| 类型 | 是什么 | 怎么用 |
|------|--------|--------|
| **Skill** | Markdown 指令文档 | 能力库告诉你路径，AI 自己读文件 |
| **MCP** | 可连接、可调用的工具服务 | 能力库负责连接 MCP 并执行工具调用 |

能力库 **不绑定** 任何 AI 平台（Hermes、Claude Code、Codex 通用）。主 AI 只要能运行命令行和读取文件，就能使用能力库。

---

## 3. 配置说明

### 3.1 config.local.json

唯一的配置文件，git 已忽略不会提交。只包含 DeepSeek API 配置。MCP 的 Key 和 Token 直接写在 `mcps/registry.json` 中。

### 3.2 MCP 通用格式

注册表在 `mcps/registry.json`：

```json
{
  "id": "mcp_xxxxxxxxxxxx",
  "name": "示例 MCP",
  "description": "这个 MCP 是什么、有哪些工具、适合什么任务",
  "remark": "使用建议、限制、注意事项",
  "enabled": true,
  "transport": "stdio",
  "params": {
    "command": "python",
    "args": ["server.py"],
    "env": {},
    "cwd": null
  }
}
```

支持两种 `transport`：

- `stdio` — 本地命令行 MCP，需要 `command`、`args`
- `streamable_http` — HTTP MCP，需要 `url`

## 4. 日常使用流程

### 4.1 三段式调用

```
主 AI 收到用户任务
    │
    ▼
① ability_suggest.py "需求描述"
    │  返回推荐的 Skill 路径 + MCP 名称
    │
    ▼
② 读 Skill 文件（直接读 Markdown）
   或 load_mcp.py --name "MCP名称"
    │  返回该 MCP 的完整工具列表
    │
    ▼
③ use_tool.py --mcp "MCP名称" --tool "工具名" --params-json "{...}"
    │  返回工具执行结果
    │
    ▼
主 AI 根据结果继续完成用户任务
```

### 4.2 常用命令

```powershell
# 咨询：问能力库"我该用什么"
python tools/advisor/ability_suggest.py "我需要搜索最新行业报告并做深度分析"

# 加载：查看某个 MCP 有哪些工具
python tools/mcp/load_mcp.py --name "tavily"

# 调用：执行具体工具
python tools/mcp/use_tool.py --mcp "tavily" --tool "tavily_search" --params-json "{\"query\":\"AI最新动态\",\"max_results\":3}"

# 传复杂参数时用文件（避免命令行转义问题，见 6.2 节）
python tools/mcp/use_tool.py --mcp "codegraph" --tool "codegraph_search" --params-file "params.json"
```

### 4.3 Session 机制

`load_mcp` 默认保持连接（`--no-keep-alive` 可关闭）。

- **stdio MCP**：load 时启动一个独立的 relay 子进程代理 MCP 通信，session 文件记下 relay 地址。后续 `use_tool` 自动通过 relay 调用，同一 MCP 的多次 tool call 共享同一个子进程和状态（如浏览器页面）。
- **HTTP MCP**：load 时缓存 session_id，后续 use 复用以省去 initialize 握手。

再次 load 同一个 MCP 时自动杀掉旧 relay/session 起新的，保证热加载。Session 文件存在系统临时目录（`%TEMP%/capability-library/sessions/`），不污染项目目录。

## 5. 能力入库

### 5.1 注册 Skill

1. 在 `skills/` 下创建独立目录，放入 `SKILL.md`
2. 运行注册命令

```powershell
python tools/registry/register_skill.py \
  --name "名称" \
  --path "skills/example/SKILL.md" \
  --description "描述" \
  --remark "备注"
```

注册后自动更新 `skills/manifest.json` 和 `skills/manifest.md`。

### 5.2 注册 MCP

**stdio 类型：**

```powershell
python tools/registry/register_mcp.py \
  --name "新 MCP" \
  --description "描述" \
  --remark "备注" \
  --transport stdio \
  --params-json "{\"command\":\"python\",\"args\":[\"server.py\"],\"env\":{},\"cwd\":null}"
```

**streamable_http 类型：**

```powershell
python tools/registry/register_mcp.py \
  --name "新 MCP" \
  --description "描述" \
  --remark "备注" \
  --transport streamable_http \
  --params-json "{\"url\":\"https://api.example.com/mcp\",\"headers\":{\"Authorization\":\"Bearer 你的Token\"}}"
```

注册后自动更新 `mcps/registry.json` 和 `mcps/registry.md`。

### 5.3 手动刷新清单

```powershell
python tools/registry/refresh_manifest_md.py
```

## 6. 实操注意事项

### 6.1 stdio MCP 的命令路径（Windows 特别重要）

Python subprocess 在 Windows 下找不到 npm 全局命令和 bash PATH 里的可执行文件。必须使用 **Windows 原生路径**：

```json
// ❌ 不行 — Windows subprocess 找不到
{ "command": "codegraph", "args": ["serve", "--mcp"] }

// ✅ 可以 — 用 cmd 扩展名 + 完整路径
{ "command": "C:\\Users\\用户名\\AppData\\Roaming\\npm\\codegraph.cmd", "args": ["serve", "--mcp"] }
```

查找可执行文件的 Windows 路径：

```powershell
where codegraph    # 返回 C:\Users\...\npm\codegraph.cmd
where npx
```

`python` 命令不需要完整路径（Python 安装时会加入系统 PATH）。

### 6.2 --params-json 的转义问题

在 bash 环境下传含 Windows 路径的 JSON 时，`\\` 双反斜杠会出问题。**避免使用 `--params-json` 传 Windows 路径**，改用 `--params-file`：

```powershell
# ❌ bash 下 `\\\\` 转义易出错
python tools/mcp/use_tool.py ... --params-json "{\"projectPath\":\"D:\\\\Github\\\\project\"}"

# ✅ 写入文件再传入
echo '{"projectPath":"D:\\Github\\project"}' > params.json
python tools/mcp/use_tool.py ... --params-file "params.json"
```

同样，注册 stdio MCP 时如果 args 包含 Windows 路径，也优先用 `--params-file`。

PowerShell 环境下单层引号包裹即可，转义问题较少。

### 6.3 codegraph MCP 需要指定项目路径

codegraph 启动后不感知当前工作目录，每次调用工具时需传 `projectPath`：

```json
{ "query": "load_mcps", "projectPath": "D:\\Github\\项目名" }
```

或者在注册时把 `--path` 加到 args 里：

```json
{ "command": "C:\\Users\\...\\codegraph.cmd", "args": ["serve", "--mcp", "--path", "D:\\Github\\项目名"] }
```

### 6.4 思源笔记 MCP

思源的两个 MCP 都是 stdio 类型，需要先启动思源笔记桌面端才能使用内核 API。MCP 进程本身可以启动，但工具调用会报 `kernel_unreachable`。

思源的 Token 直接写在 registry.json 的 env 字段中。

### 6.5 anysearch extract 工具有代理限制

免费账号的 extract 工具可能因代理资源耗尽而失败，搜索类工具不受影响。如需要大量 extract，可考虑用 tavily extract 替代。

### 6.6 timeout 参数

部分 MCP 初始化较慢（如首次 npm 执行），必要时加 `--timeout`：

```powershell
python tools/mcp/load_mcp.py --name "codegraph" --timeout 60
```

## 7. 从 AI 平台迁移能力

把 MCP 和 Skill 从 Hermes、Claude Code 等平台迁入能力库后，可以直接从平台配置中删掉，减轻平台负担。

### 7.1 从 Hermes 迁移 MCP

Hermes 的 MCP 配置在 `C:\Users\用户名\AppData\Local\hermes\config.yaml` 的 `mcp_servers` 段。

以 anysearch 为例，Hermes 配置：

```yaml
mcp_servers:
  anysearch:
    enabled: true
    headers:
      Authorization: Bearer as_sk_xxxx
    url: https://api.anysearch.com/mcp
```

转为能力库格式：

```json
{
  "name": "anysearch",
  "transport": "streamable_http",
  "params": {
    "url": "https://api.anysearch.com/mcp",
    "headers": {
      "Authorization": "Bearer 你的AnySearch Key"
    }
  }
}
```

迁移步骤：

1. 将 API Key/Token 直接填入 MCP 的 params（url、headers、env 等字段）
2. 运行 `register_mcp.py` 注册
3. 在 Hermes `config.yaml` 中删除或 disable 该 MCP

### 7.2 从 Hermes 迁移 Skill

Hermes 的 skills 目录在 `C:\Users\用户名\AppData\Local\hermes\skills\`。每个 skill 是一个带 `SKILL.md` 的目录。

迁移步骤：

1. 复制 skill 目录到能力库的 `skills/` 下
2. 运行 `register_skill.py` 注册

### 7.3 从 Claude Code 迁移 MCP

Claude Code 的 MCP 可通过 VS Code 扩展配置或 `mcpServers` 字段（在 settings.json 或 `.mcp.json` 中）。

### 7.4 从 Claude Code 迁移 Skill

Claude Code skills 通常在 `~/.claude/skills/` 下。

迁移步骤同上：复制 → 注册。注意 Skill 文件中的 YAML frontmatter（`---` 包裹的 `name`/`description`）在能力库中由注册命令单独管理，可保留也可去掉。

### 7.5 迁移后验证

```powershell
# 确认入库成功
python tools/mcp/load_mcp.py --name "迁移的MCP名称"

# 确认 Skill 清单
cat skills/manifest.json
```

迁移完成后，可在 AI 平台中禁用或删除对应配置。主 AI 通过能力库命令行间接使用这些能力，不再由平台直接管理 MCP 连接和 Skill 注入。
