# 能力库产品架构

## 定位

能力库是一个本地纯文件的能力管理层，让不同 AI 平台共用同一套 Skill 和 MCP。

它只做三件事：

1. 给 AI 初始上下文提供能力地图（CAPABILITY.md）
2. 保存 Skill 文档（SKILL.md）
3. 保存 MCP 注册配置并提供 load/use 调用工具

不替代 Agent runtime，不负责推理、调度和权限决策。

## 核心对象

能力库只有三类对象：

### CAPABILITY.md —— 能力地图

- 形态：根目录 Markdown 文件
- 读者：AI 初始上下文
- 内容：Skill 入口和摘要、MCP 简表、使用命令、维护入口
- 不包含：完整 Skill 正文、MCP 工具 schema、维护细则

### SKILL.md —— 操作指南

- 形态：`skills/<name>/SKILL.md`
- 读者：AI 按需读取
- 内容：触发场景、输入要求、执行流程、验证方式
- 入口索引：统一写在 CAPABILITY.md 中，不单独维护 manifest

### mcps/registry.json —— MCP 注册表

- 形态：根目录 JSON 文件
- 读者：`load_mcp.py` 和 `use_tool.py` 程序
- 内容：MCP 的 id、name、transport、params
- 人类可读摘要：写在 CAPABILITY.md 的 MCP 简表中

用户实际使用的版本：

```
CAPABILITY.md

  capability-library-maintenance/SKILL.md

mcps/
  registry.json
tools/
  mcp/
```

## 两条流程

### 日常使用

```
AI 读取 CAPABILITY.md
  → 需要 Skill：读取对应 SKILL.md
  → 需要 MCP：load_mcp.py 获取工具描述 → use_tool.py 调用
```

`load_mcp.py` 默认使用一次性连接，不留下后台进程。需要连续复用 stdio MCP 时显式传入 `--keep-alive`；保活 relay 默认在空闲 300 秒、调用平台 Owner 退出或收到 `--close` 时结束。Windows 下默认使用 Job Object 关闭 MCP 及其派生进程，以上策略均可通过命令行参数调整。

### 维护能力库

```
读取 skills/capability-library-maintenance/SKILL.md
  → 修改 Skill 文档或 mcps/registry.json
  → 同步更新 CAPABILITY.md
  → MCP 变更后运行 load/use 测试
```

## 设计决策

| 决策 | 理由 |
|------|------|
| 不要 advisor | CAPABILITY.md 已前置能力地图，主 AI 可自行判断，无需中间 LLM |
| 不要入库脚本 | Skill 是手写 Markdown，MCP 配置是手写 JSON，脚本不增值 |
| 不要 manifest | CAPABILITY.md 就是 Skill 索引，registry.json 就是 MCP 索引，加中间层只会漂移 |
| MCP 必须先 load 再 use | MCP 工具 schema 和可用性只能通过实际连接确认，不能静态描述 |
| 用户版 CAPABILITY.md 不可覆盖 | 它是用户的能力清单，不是程序模板 |
