# 能力库使用说明

> 读者：每次新会话启动后的 AI。
> 阶段：能力库已经安装，并已写入全局记忆或启动上下文。
> 目标：让 AI 在处理任务前主动发现和调用本地 Skill/MCP。

## 1. 你必须知道的事

能力库保存了用户长期积累的 Skill 和 MCP。它包含专门工作流、私人知识入口、本地工具、网页搜索、代码理解、笔记系统、文档处理、行业研究等能力。【这里应该替换为能力库中的实际涵盖范围】

在处理非简单任务前，先咨询能力库中是否已有相关能力。不要只依赖你自己的通用知识。

能力库根目录：

```text
<能力库根目录>
```

## 2. 命令运行规则

所有命令都先进入能力库根目录：

```bash
cd "<能力库根目录>"
```

Python 命令按系统选择：

- Windows：优先用 `python`，不可用时试 `py -3`
- macOS / Linux：优先用 `python3`，不可用时试 `python`

下面示例统一写作 `python`。如果当前系统不是这个命令，请替换成可用的 Python 命令。

## 3. 先查询该用什么能力

每次遇到非简单任务，先运行：

```bash
python tools/advisor/ability_suggest.py "<描述你收到的任务需求>"
```

示例：

```bash
python tools/advisor/ability_suggest.py "用户想搜索一家公司的最新资料，并写一份研究报告"
```

advisor 会返回 JSON，通常包含：

- `skills`：建议读取的 Skill
- `mcps`：建议加载的 MCP
- `summary`：简短建议

## 4. 使用 Skill

如果 advisor 返回 Skill：

1. 根据实际路径，读取返回项中的 `SKILL.md`。
2. 按该 Skill 的流程执行任务。

## 5. 使用 MCP

如果 advisor 返回 MCP，先加载MCP：

```bash
python tools/mcp/load_mcp.py --name "<MCP名称或ID>"
```

读取返回的工具列表和参数 schema 后，再调用具体工具（必须先加载再调用，否则会报错）：

```bash
python tools/mcp/use_tool.py --mcp "<MCP名称或ID>" --tool "<工具名>" --params-json "{...}"
```

参数复杂导致命令传递失败时，在tools/mcp写入params.json，使用 JSON 文件传递参数：

```bash
python tools/mcp/use_tool.py --mcp "<MCP名称或ID>" --tool "<工具名>" --params-file params.json
```

`load_mcp.py` 默认保持会话，后续 `use_tool.py` 会复用连接。

## 6. advisor 失败时

如果 advisor 因 API Key、网络、模型配置或其他原因失败：

1. 告诉用户失败原因。
2. 直接读取 `skills/manifest.json` 和 `mcps/registry.json`。
3. 根据 `description` 手动选择相关 Skill/MCP。

不要因为 advisor 失败就忽略能力库。

## 7. 什么时候不需要查询能力库

- 用户明确要求不要使用工具或能力库

除此之外，都必须查询能力库。
