# 能力库设计思路

能力库是一个本地、纯文件、跨 Agent 平台的能力注册与调用层。它不依赖任何 Agent 框架，不要求启动时把所有能力注入上下文。

## 核心思想

能力库不把 Skill 和 MCP 混成一种东西，而是在推荐层统一管理，在使用层分别处理。

- Skill 是 Markdown 指令文档。能力库返回路径，主 AI 使用平台自己的文件读取能力读取。
- MCP 是可连接、可列工具、可调用的工具服务。能力库负责加载 MCP 并调用工具。

## 目录结构

```text
skills/
  manifest.json
  manifest.md
  capability-library-guide/SKILL.md
  intake-flow/SKILL.md

mcps/
  registry.json
  registry.md

tools/
  advisor/ability_suggest.py
  mcp/load_mcp.py
  mcp/use_tool.py
  registry/register_skill.py
  registry/register_mcp.py
  registry/refresh_manifest_md.py
```

## 运行流程

1. 用户在任意 Agent 平台发起任务。
2. 主 AI 判断可能需要额外能力。
3. 主 AI 调用 `ability_suggest.py`，传入自然语言需求。
4. 前台咨询 LLM 读取能力库说明、Skill 清单和 MCP 注册表。
5. 前台咨询返回建议的 Skill、MCP 和简短原因。
6. 程序整理为干净 JSON，返回给主 AI。
7. 主 AI 读取 Skill 文件，或加载 MCP 获取工具描述。
8. 主 AI 调用 MCP 工具，能力库执行后返回结果。

## 入库流程

Skill 入库：

1. 将 Skill 文件放入 `skills/` 下的独立目录。
2. 调用 `register_skill.py`，传入名称、路径、描述、备注。
3. 注册工具检查路径和重复项，生成随机 ID，写入清单。
4. 自动刷新 `skills/manifest.md`。

MCP 入库：

1. 将 MCP 配置整理为能力库统一格式。
2. 调用 `register_mcp.py`，传入名称、描述、备注、传输类型和参数。
3. 注册工具检查 JSON、传输类型和必填字段，生成随机 ID，写入注册表。
4. 自动刷新 `mcps/registry.md`。

## MCP 统一格式

MCP 注册项同时给前台咨询阅读，也给加载和调用程序执行。

```json
{
  "id": "mcp_随机十六进制",
  "name": "示例 MCP",
  "description": "较长描述，说明这个 MCP 是什么、有哪些主要工具、适合什么任务。",
  "remark": "使用建议、限制、注意事项。",
  "enabled": true,
  "transport": "stdio",
  "params": {
    "command": "node",
    "args": ["server.js"],
    "env": {},
    "cwd": null
  }
}
```

支持的传输类型：

- `stdio`：本地命令行 MCP。
- `streamable_http`：远程或本地 HTTP MCP。

## 第一版边界

- 不绑定任何 Agent 平台。
- 不做远程能力库，只保留未来参数位置。
- 不自动翻译 Claude Code、Codex、Hermes 等外部 MCP 配置格式。
- 不维护 MCP 常驻进程，`load_mcp` 和 `use_tool` 每次独立重连。
