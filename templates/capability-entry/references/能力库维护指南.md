# 能力库维护指南

## 维护边界

普通用户 Skill 放在平台共享的用户 Skill 目录；低频成组能力和按需 MCP 放在能力库。新增或修改前先检查是否已有相同能力。

## 修改 Skill

维护对应 `skills/<name>/SKILL.md`，并同步更新 `capability-entry/SKILL.md` 中的能力地图。

## 修改 MCP

维护 `mcps/registry.json` 和入口中的 MCP 摘要。修改后先 load 获取实时工具描述，再执行安全的最小 use；保活测试结束后必须显式 close。

## 删除规则

删除 Skill、MCP、脚本或旧索引前必须获得用户明确确认，并先检查入口、注册表和平台链接是否仍有引用。
