# 能力库（Capability Library）

## 概念

能力库是一个完全独立的、纯文件组成的能力注册中心。不依赖任何 Agent 框架。

AI Agent 启动时零预加载——不往 context 里注入任何能力列表。只给两个工具入口：search 和 read。

## 能力库包含什么

目录结构示意：

capability-library/
  skills/       -- SKILL.md 格式的行为指令文档
  tools/        -- MCP 注册 JSON、命令行封装脚本、工具描述
  indexes/      -- 可选的预检索引擎数据

每项能力就是一个文件。.md 是技能指令，.json 是工具注册信息，.py/.sh 是可执行入口。

## 检索引擎

内置，两种模式共用同一入口：

search(query, mode="auto")

- direct 模式：关键词、FTS5、正则表达式直接搜索文件内容，返回匹配片段
- llm 模式：用 embedding 做语义匹配，或让 LLM 把自然语言需求翻译成查询去搜
- auto 模式（默认）：根据 query 内容和匹配情况自动选择或组合

返回格式统一：能力名、类型（skill/tool）、路径、匹配片段、相关性分数。

AI 拿到结果后调 read(name) 加载完整内容。

## 检索范围

- 本地：搜索 skill 目录下所有文件，基于 ripgrep/全文索引
- 云端（扩展）：联网拉取远程仓库索引或搜索社区能力库

同一套 search 接口覆盖两种范围，通过参数控制 scope="local" | "remote" | "all"。

## 能力来源

- 本地 skill 目录里的 .md 文件
- 本地 tools 目录里的 .json/.py/.sh 文件
- 远程仓库/社区发布的能力索引

能力文件可以手动编写，也可以由 AI 生成保存（类似 Hermes 的 skill_manage）。

## 关联概念

### MCP 动态加载

能力库不直接管理 MCP 连接。MCP 的按需加载通过 mcp-cli 模式处理：

- tools/ 目录下的 .json 文件是 MCP 注册信息
- 能力库的 search 可以搜到这些 MCP 注册信息
- 实际连接由外部工具（mcp-cli 或同类）完成

### 与 Hermes 现有 skill 系统的关系

Hermes 当前做法：启动时把所有 skill 名字+描述注入 system prompt（progressive disclosure 但仍预加载索引）。

能力库做法：零预加载。AI 只知道有 search 和 read 两个工具可用。遇到任务时先 search 找到相关能力，再 read 按需加载。

兼容性：能力库的 skills/ 目录可以是 Hermes skills/ 目录的超集或镜像，格式互通。

## 社区已有实践

- mcp-cli（philschmid/mcp-cli）：1176 star，CLI 方式动态发现和调用 MCP，按需 info/call，上下文消耗从 47000 tokens 降到 400
- SkillFlow（arXiv 2504.06188）：多阶段技能检索流水线，从 36000 个社区 SKILL.md 中检索
- SRA-Bench（arXiv 2604.24594）：正式定义 Skill Retrieval Augmentation 范式，26000+ 技能基准测试集

目前没有项目把 skill 检索和 MCP 动态加载统一到一个入口。

## 期望效果

- AI Agent 可以拥有无限能力但零 context 开销
- 能力可以热更新（增删改文件即可，不需重启）
- 不绑定任何 Agent 框架，纯文件协议
- 本地和云端能力统一检索
- skill 和 工具 在同一个检索入口下一视同仁
